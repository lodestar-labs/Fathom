Fathom · Documentation

Export data of any shape out of a relational database — declaratively

Describe an export once in a small JSON definition — entities, fields, hierarchy, filters, code lookups — and Fathom streams it out of SQL Server as JSON, XML, or CSV, however deep the nesting, without a line of export code.

What Fathom does

Every system that imports data eventually has to give it back: a partner wants XML, an analyst wants CSV, another service wants JSON. Fathom is the one export pipeline for all of those.

You register an export definition — a JSON document describing a root entity, its child entities, which columns become which output fields, and which filters callers may use. From then on, the export is a single HTTP call:

GET /api/exports/orders/run?format=xml&country=Denmark&orderDateFrom=2026-01-01

Fathom validates the filters, stages the data level by level, and streams the response — it never buffers the whole export in memory, so a ten-million-row export costs the same memory as a ten-row one. Reference-data codes (a numeric country or species code) are rendered as readable strings on the way out through pluggable lookups, and the same lookups work in reverse so callers can filter by the readable value.

JSON

Roots as an array; each entity's children nested under a property named after the child entity.

XML

One element per entity, fields as child elements, database NULL as nil="true".

CSV

One file per entity in a zip; rows carry _key / _parentKey so the hierarchy can be rebuilt — or re-imported by Loadstone.

Quick start

# 1. Run it (SQL Server + Fathom, seeded with the orders sample)
git clone https://github.com/lodestar-labs/Fathom.git
cd Fathom
docker compose up

# 2. See what's registered
curl http://localhost:8080/api/exports

# 3. Run an export
curl "http://localhost:8080/api/exports/orders/run?format=json&country=Denmark"

From zero to a filtered, hierarchical, streamed export in three commands. The Using Fathom guide continues from here.

Fathom and Loadstone

Fathom is the conceptual inverse of Loadstone: Loadstone imports data of any shape into a relational database from a declarative manifest; Fathom exports it back out from a declarative definition. They share design DNA — API-first, streaming, declarative JSON at the boundary — and Fathom's hierarchical CSV output (_key/_parentKey) is designed so a Fathom export can be fed straight back through a Loadstone import. The round-trip is worked end-to-end in Using Fathom §8.
